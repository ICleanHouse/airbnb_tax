# Project requirements overview
#
# Python dependencies are installed through this file by including the backend
# requirements below:
#
#   pip install -r requirements.txt
#
# Non-Python technologies required by the project:
#
# - Python 3.13+ for the Django backend
# - Node.js 22+ and npm 11+ for the Next.js frontend
# - PostgreSQL 16 for production/local Docker database
# - Redis 7 for Celery broker/result backend, cache, locks, and temporary state
# - Docker 24+ and Docker Compose for the full local stack
#
# Frontend libraries (managed in frontend/package.json):
#
# - Next.js 15.5+
# - React 19.2+
# - lucide-react 0.468+
# - TypeScript 5.9+
# - ESLint 9.39+
# - eslint-config-next 15.5+
#
# Backend Python libraries (managed in backend/requirements.txt):
#
# - Django 6.0+
# - djangorestframework 3.17+
# - psycopg 3.2+ (PostgreSQL driver, Docker only)
# - celery 5.4+ (async task queue, Docker only)
# - redis 5.2+ (broker/cache client, Docker only)
# - django-cors-headers 4.6+
# - gunicorn 23.0+ (production WSGI server, Docker only)
# - icalendar 6.3+ (iCal / Airbnb calendar parsing)
# - python-dotenv 1.2+ (auto-load .env in development)
# - Pillow 11+ (Django image uploads)
# - sentry-sdk 2.20+ (Django/Celery crash monitoring, enabled by SENTRY_DSN)
#
# Infrastructure/config files:
#
# - docker-compose.yml for local PostgreSQL, Redis, backend, worker, and frontend
# - .env.example for local environment variables
# - backend/requirements.txt for backend Python packages
# - frontend/package.json for frontend Node packages
